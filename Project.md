# 选课推荐系统 — 项目验收说明书

> 项目名称：选课推荐系统  
> 版本：v1.0  
> 编写日期：2026-05-08

---

## 一、项目概述

选课推荐系统是一个面向高校的课程管理与选课辅助平台，采用前后端分离架构，提供课程检索、教师评价、模拟选课、账户管理等一站式服务。系统支持超级管理员、管理员、学生三级权限，兼顾桌面端与移动端体验。

---

## 二、功能验收清单

### 2.1 认证模块

| 编号 | 功能 | 说明 | 角色 |
|------|------|------|------|
| A-01 | 登录 | 用户名 + 密码登录，JWT Token 认证，有效期 30 天 | 全部 |
| A-02 | 注册 | 填写用户名、密码、真实姓名、学号、申请角色 | 全部 |
| A-03 | 注册审核 | 超级管理员审核通过/拒绝注册申请 | 超级管理员 |
| A-04 | Token 刷新 | 自动刷新 Token，保持登录状态 | 全部 |

### 2.2 学生端

| 编号 | 功能 | 说明 |
|------|------|------|
| S-01 | 课程检索 | 搜索课程名，实时筛选 |
| S-02 | 专业筛选 | 按专业分类筛选课程 |
| S-03 | 课程性质筛选 | 支持多选课程性质（通识必修、专业选修等 9 种） |
| S-04 | 必修/选修筛选 | 选择专业后，按该专业的必修/选修筛选 |
| S-05 | 教师搜索 | 支持拼音缩写搜索（如 `wls` → 汪礼胜） |
| S-06 | 开课学院筛选 | 按开课学院筛选 |
| S-07 | 课程详情 | 查看课程详细信息（学分、教师、学期、课程性质等） |
| S-08 | 教师评价 | 查看教师评分（按老师分组，显示评分和评价人数） |
| S-09 | 添加评价 | 选择教师进行评分和评价 |
| S-10 | 删除评价 | 删除自己的评价 |
| S-11 | 建议修读学期 | 显示课程对当前专业的建议修读学期 |

### 2.3 管理员端

| 编号 | 功能 | 说明 |
|------|------|------|
| M-01 | 双模式切换 | 默认学生模式，右上角一键切换管理面板 |
| M-02 | 课程列表 | 查看所有课程，支持分页、排序 |
| M-03 | 添加课程 | 设置课程名、学分、学院、教师、学期、课程性质等 |
| M-04 | 编辑课程 | 修改课程信息 |
| M-05 | 删除课程 | 删除指定课程 |
| M-06 | 多学期支持 | 一门课可同时选择春季/秋季 |
| M-07 | 多课程性质 | 一门课可同时属于多个课程性质 |
| M-08 | 分专业设置 | 为不同专业设置不同的必修/选修性质 |
| M-09 | 课程序号 | 自定义序号，支持升序/降序排列 |
| M-10 | 评价管理 | 查看、添加、编辑、删除所有评价 |
| M-11 | 评价搜索 | 支持课程名搜索和教师搜索 |
| M-12 | 教师检索 | 按教师查看课程，支持拼音缩写搜索 |

### 2.4 超级管理员

| 编号 | 功能 | 说明 |
|------|------|------|

| T-02 | 创建账户 | 创建管理员或学生账户 |
| T-03 | 审核注册 | 审核通过/拒绝注册申请 |
| T-04 | 修改角色 | 修改用户角色 |

| T-06 | 解密查看 | 解密查看完整的学号和姓名 |
| T-07 | 数据清空 | 清空所有课程和评价数据 |

### 2.5 模拟选课

| 编号 | 功能 | 说明 |
|------|------|------|
| C-01 | 专业选择 | 选择专业（计算机类、软件、大数据等 6 个） |
| C-02 | 方向选择 | 计算机类需选择分流方向（计算机/软件） |
| C-03 | 班级选择 | 选择普通班/卓越工程师 |
| C-04 | 通识必修 | 默认全选 |
| C-05 | 英语选课 | 快班/慢班选择，需修满 8 学分 |
| C-06 | 体育选课 | 基础体育 + 选修，需修满 4 学分，检测先修关系 |
| C-07 | 学科必修 | 按专业加载，默认全选 |
| C-08 | 专业必修 | 按专业加载，默认全选 |
| C-09 | 专业选修 | 按专业加载，需达到学分要求 |
| C-10 | 实践课 | 按专业加载，默认全选 |

| C-12 | 个性课程 | 需修满 6 学分 |
| C-13 | 选课总结 | 汇总已选课程，支持导出为文本文件 |
| C-14 | 学分看板 | 实时统计已选学分 |

## 五、部署架构

```
用户 → Nginx (80端口) → Gunicorn (2 workers) → MySQL + Redis
         ├── 静态资源（JS/CSS/图片）→ 直接返回（强缓存）
         └── /api/* → 反向代理到后端
```
